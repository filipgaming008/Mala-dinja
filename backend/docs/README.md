# docs

Documentation hub for backend planning, delivery tracking, contracts, and release readiness.

## documentation update rule

For every implementation prompt, update the relevant file in `docs/` in the same change.

- `docs/context/`: product context, architecture, and implementation status
- `docs/contracts/api/`: endpoint source-of-truth contracts
- `docs/delivery/`: roadmap and task execution tracking
- `docs/release/`: runbook and readiness checks
